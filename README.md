# Design Shorts

Design Shorts is a zero-cost MVP for mobile-first HLD prep. It delivers one short topic per screen, remembers progress locally, and keeps a separate tech news digest backed by a free GitHub Actions pipeline.

For fast project handoff and build/content context, read [DESIGN.md](/Users/deepthi/VinayProj/design-shorts/DESIGN.md) first.
For product direction and progress tracking, read [PRODUCT_PATH.md](/Users/deepthi/VinayProj/design-shorts/PRODUCT_PATH.md).
For tactical implementation tasks, read [EXECUTION_PLAN.md](/Users/deepthi/VinayProj/design-shorts/EXECUTION_PLAN.md).
For visual and typography direction, read [UI_DIRECTION.md](/Users/deepthi/VinayProj/design-shorts/UI_DIRECTION.md).

## MVP features

- Swipe-based HLD feed
- `Serial` mode with checkpoint resume
- `Random` mode with `75% unread / 25% read` weighting
- Expandable article mode for deeper reading
- Separate tech news feed
- Local-first storage with offline HLD content
- Modular remote content sync using catalog, book, and chapter files

## Stack

- Expo React Native
- TypeScript
- AsyncStorage for local checkpoints and settings
- Static `public/news.json` digest generated by GitHub Actions
- Modular knowledge content from `src/content/catalog.json` and chapter files

## Run locally

```bash
npm install
npm start
```

Create a local env file if you want the app to fetch the live tech news digest and remote knowledge content instead of only bundled/cached content:

```bash
cp .env.example .env
```

## Mobile deployment

The app is prepared for Expo Application Services with:

- iOS bundle identifier: `com.app.designshorts`
- Android package: `com.app.designshorts`
- EAS build profiles in [eas.json](/Users/deepthi/VinayProj/design-shorts/eas.json)

Recommended release flow:

```bash
npm install
npx expo login
npx eas login
npx eas build:configure
npx eas build --platform android --profile preview
npx eas build --platform ios --profile preview
```

For production builds:

```bash
npx eas build --platform android --profile production
npx eas build --platform ios --profile production
```

After the first successful store-connected setup, submit with:

```bash
npx eas submit --platform android --profile production
npx eas submit --platform ios --profile production
```

Notes:

- `preview` is best for internal device testing
- `production` is for Play Store / App Store release builds
- `npx eas build:configure` will create any missing Expo-managed native build metadata in your account
- you will still need your own App Store Connect and Google Play Console accounts for store submission

## News digest pipeline

The app fetches a static JSON digest from a hosted URL and falls back to bundled or cached data if that fetch fails.

1. Set `EXPO_PUBLIC_NEWS_DIGEST_URL` in `.env` to your GitHub raw JSON URL.
2. Push this repo to GitHub.
3. Enable the `Refresh News Digest` workflow.
4. The workflow writes `public/news.json` on a schedule.

You can also generate the file locally:

```bash
npm run generate:news
```

## Knowledge content architecture

Knowledge content is intentionally split into small modular files instead of one large topics payload:

- `src/content/catalog.json`
- `src/content/books/<book-id>/book.json`
- `src/content/books/<book-id>/chapters/chapter-xx.json`

Why this structure exists:

- keeps content maintainable as chapters grow
- allows chapter files to be fetched in parallel
- makes caching granular
- supports future book/chapter browsing without redesigning the storage model

The app merges the content locally into a single feed:

1. load bundled fallback content
2. load cached catalog/book/chapter files if available
3. refresh remote content from `EXPO_PUBLIC_CONTENT_BASE_URL`
4. assemble the final swipe feed in the app

This means users can install the app once and receive future chapter additions without requiring a full app reinstall, as long as the remote content files are updated.
